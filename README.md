#### 🔹 프로젝트 스토리
- **기획 배경**: 목표 공유와 응원을 위한 교환일기 시스템  
- **개발 과정**: 도메인 설계 → DB 설계 → 기능 구현 → 테스트 및 리팩토링  
- **성과 요약**: 자료구조 선택, 뷰 흐름 문제 등 실전 트러블슈팅을 통해 문제 해결력과 설계 감각을 키움

#### 🔹 기술스택
- Java 11 / Spring Boot 2.7.14 / Spring Security 5.7.10  
- Thymeleaf 3.0.0 / Lombok 1.18.28 / OpenAI API 0.31.0 / jQuery 3.7.1

#### 🔹 트러블슈팅 사례

<div style="background-color: #fff9e6; padding: 20px; border-radius: 8px; font-family: Arial, sans-serif;">

**1. Set 자료구조로 인한 순서 꼬임**  
- 문제: Set은 삽입 순서를 보장하지 않아 일기 순서가 뒤섞임  
- 분석: Java의 HashSet은 해시 기반 저장 구조로 순서 무관  
- 해결: List로 교체 + currentTurn, lastTurnDate 변수 도입  
- 결과: 순차적 흐름 보장 → 사용자 경험 개선

**2. 목표 달성율 계산 누락**  
- 문제: 수정된 목표 상태가 즉시 반영되지 않음  
- 분석: 트랜잭션 커밋 이전에 UI가 렌더링됨 → 비동기 처리 누락  
- 해결: 수정 이벤트 발생 시 명시적 재계산 로직 추가  
- 결과: 데이터 일관성 확보 → 신뢰도 향상

**3. Redirect 후 데이트피커 값 사라짐**  
- 문제: PRG(Post-Redirect-Get) 패턴으로 인해 값 유지 실패  
- 분석: GET 요청 시 서버 상태 초기화 → 클라이언트 상태 미전달  
- 해결: 쿼리 파라미터 전달 + 조건부 렌더링  
- 결과: UI 상태 복원 → UX 안정화

**4. 템플릿 코드 중복 증가**  
- 문제: 조건별 버튼 표시로 fragment 분리 → 유지보수 어려움  
- 분석: View Layer에서 조건 분기 로직이 분산됨  
- 해결: `th:if` 조건 렌더링으로 통합  
- 결과: 코드 재사용성 향상 → 유지보수 효율 증가

</div>

#### 🔹 협업 및 소감
- **역할**: 교환일기 로직, 목표 달성율 계산, 감정 요약 기능 구현  
- **협업 경험**: 기능 흐름 공유, 코드 병합 및 테스트 주도  
- **향후 다짐**: 흐름을 이해하고 구조를 설계하는 개발자로 성장하고 싶음

#### 🔹 트러블슈팅 사례
<div style="background-color: #fff9e6; padding: 20px; border-radius: 8px; font-family: Arial, sans-serif;">
**1. 신고 콘텐츠 새로고침 후에만 블라인드 처리됨**

- 해결: `_.groupBy()`로 신고 항목 그룹화 + 누적 수 시각화  
- 결과: 운영자 판단 기준 강화 → 관리 효율 향상

**3. Redux 상태 반영 오류**

- 분석: Redux store 변경 감지 실패 → useSelector 최적화 미흡  
- 해결: useEffect 기반 상태 감지 로직 재구성  
- 결과: 상태 흐름 안정화 → UI 반응성 개선

#### 🔹 프로젝트 스토리
- **기획 배경**: 반려동물 동반 여행 추천 플랫폼 구축  
- **개발 과정**: 공공 API 분석 → DB 저장 → 여행 경로 추천 알고리즘 구현 → UI/UX 설계 및 배포  
- **성과 요약**: API 병렬 호출, 경로 추천 알고리즘, 무한스크롤 검색 기능 구현

#### 🔹 기술스택
- Java 11 / Spring Boot 2.7.14 / MyBatis 2.3.1  
- React 18.3.1 / Kakao Map API 1.2.0 / Axios 1.9.0 / dnd-kit 6.3.1

#### 🔹 트러블슈팅 사례
<div style="background-color: #fff9e6; padding: 20px; border-radius: 8px; font-family: Arial, sans-serif;">
**1. routeData.map null 오류**  
- 문제: 컴포넌트 렌더링 시 map 호출 오류 발생  
- 분석: React의 렌더링 순서상 useEffect 이전에 map 실행됨 → null 참조  
- 해결: 조건부 렌더링 (`routeData && routeData.map`) 적용  
- 결과: 컴포넌트 안정성 확보 → 오류 방지

**2. 드래그앤드롭 이벤트 충돌 (@dnd-kit)**  
- 문제: 클릭 이벤트와 drag 이벤트 간 충돌  
- 분석: 이벤트 버블링과 캡처링 순서 충돌 → stopPropagation 필요  
- 해결: 이벤트 우선순위 조정 + 이벤트 분리 처리  
- 결과: UI 안정화 → 사용자 편의성 향상

**3. 여행 경로 추천 알고리즘 비현실적 경로 생성**  
- 문제: 단순 나열 방식으로 비효율적 경로 추천  
- 분석: 사용자 선호도, 거리, 리뷰 수 등 다양한 요소 미반영  
- 해결: 거리 기반 BFS 알고리즘 + 가중치 설계로 현실적 경로 제공  
- 결과: 사용자 만족도 향상 → 추천 정확도 개선

**4. 대용량 리뷰 데이터로 인한 검색 성능 저하**  
- 문제: 전체 리뷰 데이터를 한 번에 로딩 → 렌더링 부하 발생  
- 분석: 클라이언트 메모리 과부하 + 브라우저 렌더링 병목  
- 해결: LIMIT/OFFSET 기반 페이징 처리 + 무한스크롤 적용  
- 결과: 렌더링 비용 분산 → UX 최적화 및 성능 개선
</div>

#### 🔹 협업 및 소감
- **역할**: 팀장 / 여행 경로 추천, 장소 검색, 일정 저장 기능 개발  
- **협업 경험**: 일정 조율, 기능 분담, 배포 가이드 작성
